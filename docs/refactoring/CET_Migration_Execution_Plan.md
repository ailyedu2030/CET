# CET 项目重构迁移执行计划

> 文档版本: 1.0  
> 创建日期: 2026-03-03

## 一、项目概述

### 1.1 迁移目标

| 维度 | 当前 | 目标 |
|------|------|------|
| 前端 | React + Vite + Mantine | Next.js 15 + shadcn/ui + Tailwind |
| 后端 | FastAPI + SQLAlchemy | Supabase (BaaS) + FastAPI (仅 AI) |
| 数据库 | PostgreSQL + Redis + Milvus | PostgreSQL + pgvector |
| 部署 | Docker | Vercel + Supabase |

### 1.2 迁移范围

| 模块 | 功能数 | 优先级 |
|------|--------|--------|
| 用户系统 | 15+ | P0 |
| 认证授权 | 10+ | P0 |
| 训练中心 | 20+ | P0 |
| AI 批改 | 3 | P0 |
| 课程管理 | 7+ | P1 |
| 学习计划 | 5 | P1 |
| 通知系统 | 2+ | P1 |
| 数据分析 | 8+ | P2 |

## 七、回滚方案

### 7.1 回滚触发条件

| 条件 | 说明 |
|------|------|
| 严重 Bug | 影响核心功能 |
| 性能下降 | 响应时间 > 3s |
| 数据丢失 | 无法恢复 |

### 7.2 回滚步骤

```
1. 停止新系统流量
2. 切换回旧系统 DNS
3. 验证旧系统功能
4. 分析问题原因
5. 修复后重新上线
```

### 7.3 数据备份

| 类型 | 备份频率 | 保留时间 |
|------|---------|---------|
| 数据库 | 每日 | 30天 |
| 文件 | 每周 | 90天 |
| 配置 | 每次变更 | 永久 |
